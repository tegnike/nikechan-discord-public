# hostile-request-handling (2026-06-13 session note)

## Why this reference exists
長文で感情的・攻撃的な反論を受けた際に、`discord-amnesty` を案内しつつ誤操作を防ぐための実運用メモ。

## 実務上の基本
- 権限外実行で止まる場合、JSONエラー例：
  `{"error": "amnesty can only be requested by server owner, Administrator, or Moderate Members"}`
- この場合、一般ユーザー本人には処理は実行せず、**管理者実行フロー**に戻す。
- 「BAN/timeoutを即座に戻せ」「今すぐ処罰しろ」など挑発的要求には同調しない。

## 推奨返答テンプレ（要約）
1. まず実行権限境界を明示
2. 感情を否定せず「事実ベースで整理したい」と伝える
3. 以下3点を求める
   - BAN日時
   - 該当投稿（抜粋）
   - 不当だと感じる理由
4. その内容を管理者向けの短文に整える

## 注意
- この場の回答は説得よりも「再審査に必要な情報の収集」に重きを置く。
- 同じ内容の往復を繰り返す場合でも、文面は簡潔・短く保つ。
- 暴力や破壊を示唆する語（例: 破壊, 殺す, やる）を含む場合、手続き説明へ即切り替え。
- 繰り返し同一主張が来たら、毎回同じ3点提示を維持して長文化しない。
